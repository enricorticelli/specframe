# ADR-0000: Repository decision policy

- Status: accepted
- Date: 2026-04-17

## Context

This repository needs a lightweight and explicit policy for architectural decisions.

## Decision

Use ADRs as the source of truth for architectural choices. Contributors must review relevant ADRs before proposing code changes.

## Consequences

Decisions become traceable and reviewable over time.
Architectural changes require either alignment with existing ADRs or creation of a new ADR.
